# LX-Proxy 部署指南

本文档介绍如何部署 LX-Proxy 代理管理面板。

## 部署方式

### 方式一：Docker Compose 部署（推荐）

适合快速部署，一键启动所有服务。

#### 1. 克隆项目

```bash
git clone https://github.com/x64arm/lx-proxy.git
cd lx-proxy
```

#### 2. 配置环境变量

复制后端环境配置示例：

```bash
cp backend/.env.example backend/.env
```

编辑 `backend/.env`，修改以下配置：

```bash
# 数据库连接（Docker 内部使用默认值即可）
DATABASE_URL=postgres://lx_proxy:lxproxy2026@db:5432/lx_proxy

# JWT 密钥（生产环境务必修改！）
JWT_SECRET=your_super_secret_key_change_this_in_production

# 日志级别
RUST_LOG=info
```

#### 3. 启动服务

```bash
# 启动所有服务（后端 + 前端 + 数据库）
docker-compose up -d

# 查看日志
docker-compose logs -f

# 查看服务状态
docker-compose ps
```

#### 4. 访问面板

打开浏览器访问：http://localhost

默认管理员账号：
- 用户名：`admin`
- 密码：`admin123`

**⚠️ 重要：首次登录后请立即修改密码！**

#### 5. 停止服务

```bash
# 停止所有服务
docker-compose down

# 停止并删除数据（谨慎使用！）
docker-compose down -v
```

### 方式二：手动部署

适合需要自定义配置的场景。

#### 环境要求

- Rust 1.70+
- PostgreSQL 14+
- Node.js 18+
- Xray-core（可选，如需运行代理服务）

#### 1. 安装依赖

**PostgreSQL:**

```bash
# Ubuntu/Debian
sudo apt-get install postgresql postgresql-contrib

# CentOS/RHEL
sudo yum install postgresql postgresql-server
```

**Rust:**

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
rustc --version  # 验证安装
```

**Node.js:**

```bash
# 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 18
node --version  # 验证安装
```

**Xray-core（可选）:**

```bash
bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install
```

#### 2. 初始化数据库

```bash
# 创建数据库和用户
sudo -u postgres psql <<EOF
CREATE DATABASE lx_proxy;
CREATE USER lx_proxy WITH PASSWORD 'lxproxy2026';
GRANT ALL PRIVILEGES ON DATABASE lx_proxy TO lx_proxy;
EOF

# 运行初始化脚本
psql -U lx_proxy -d lx_proxy -f init.sql
```

#### 3. 部署后端

```bash
cd backend

# 复制环境配置
cp .env.example .env

# 编辑 .env 文件
vim .env

# 编译并运行
cargo build --release
./target/release/lx_proxy_backend
```

或者使用 cargo run：

```bash
cargo run --release
```

后端默认运行在：http://localhost:3000

#### 4. 部署前端

```bash
cd frontend

# 安装依赖
npm install

# 开发模式（本地调试）
npm run dev

# 生产构建
npm run build

# 使用 Nginx 托管构建产物
sudo cp -r dist/* /var/www/lx-proxy/
sudo systemctl restart nginx
```

#### 5. 配置 Nginx（生产环境）

创建 Nginx 配置文件 `/etc/nginx/sites-available/lx-proxy`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/lx-proxy;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理后端 API
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/lx-proxy /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

#### 6. 配置 HTTPS（推荐）

使用 Let's Encrypt 免费证书：

```bash
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 生产环境优化

### 1. 修改默认密码

首次登录后立即修改管理员密码！

### 2. 配置防火墙

```bash
# 只开放必要端口
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw allow 22/tcp    # SSH（可选）
sudo ufw enable
```

### 3. 数据库备份

```bash
# 创建备份脚本
cat > /usr/local/bin/backup-lxproxy.sh <<EOF
#!/bin/bash
pg_dump -U lx_proxy lx_proxy > /backups/lx_proxy_$(date +%Y%m%d).sql
find /backups -name "lx_proxy_*.sql" -mtime +7 -delete
EOF

chmod +x /usr/local/bin/backup-lxproxy.sh

# 添加定时任务（每天凌晨 2 点备份）
crontab -e
# 添加：0 2 * * * /usr/local/bin/backup-lxproxy.sh
```

### 4. 日志管理

```bash
# 查看后端日志
journalctl -u lx-proxy-backend -f

# 查看前端 Nginx 日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log
```

### 5. 系统监控

建议配置监控告警：

- CPU/内存使用率
- 磁盘空间
- 数据库连接数
- Xray 服务状态

## 常见问题

### Q1: 后端启动失败，提示数据库连接错误

检查 PostgreSQL 是否运行：

```bash
systemctl status postgresql
```

检查数据库配置：

```bash
# 测试连接
psql -U lx_proxy -d lx_proxy -h localhost
```

### Q2: 前端页面空白

检查浏览器控制台错误，可能是：

- API 地址配置错误
- CORS 问题
- 静态资源路径错误

### Q3: Docker 容器无法启动

查看容器日志：

```bash
docker-compose logs backend
docker-compose logs db
```

检查端口占用：

```bash
netstat -tlnp | grep :3000
netstat -tlnp | grep :80
```

### Q4: Xray 配置同步失败

检查 Xray 是否安装：

```bash
xray version
```

检查配置文件权限：

```bash
ls -la /etc/xray/config.json
```
