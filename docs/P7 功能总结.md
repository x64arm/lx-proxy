## 🚀 使用指南

### 批量操作使用

#### 1. 在列表中启用批量选择

```vue
<template>
  <el-table
    :data="items"
    @selection-change="handleSelectionChange"
  >
    <el-table-column type="selection" width="55" />
    
    <!-- 其他列 -->
  </el-table>
  
  <BatchActions
    v-if="selectedItems.length > 0"
    v-model:selectedItems="selectedItems"
    :all-selected="allSelected"
    @select-all="toggleSelectAll"
    @batch-enable="handleBatchEnable"
    @batch-disable="handleBatchDisable"
    @batch-delete="handleBatchDelete"
  />
</template>
```

#### 2. 批量导出配置

```javascript
// 选择要导出的配置
const selectedIds = selectedItems.value.map(item => item.id)

// 调用 API
const response = await api.post('/api/batch/inbounds/export', {
  ids: selectedIds
})

// 下载 JSON 文件
const blob = new Blob([JSON.stringify(response.data.data, null, 2)], {
  type: 'application/json'
})
const url = URL.createObjectURL(blob)
const a = document.createElement('a')
a.href = url
a.download = `lx-proxy-export-${Date.now()}.json`
a.click()
```

#### 3. 批量导入配置

```javascript
// 读取 JSON 文件
const file = event.target.files[0]
const text = await file.text()
const configs = JSON.parse(text)

// 调用 API
const response = await api.post('/api/batch/inbounds/import', {
  items: configs
})

console.log(`导入成功：${response.data.processed} 个，失败：${response.data.failed} 个`)
```

## 🔐 安全考虑

### 批量操作安全

1. **权限验证**
   - 仅管理员可执行批量删除
   - 普通用户只能操作自己的配置

2. **操作审计**
   ```rust
   // 记录批量操作日志
   for id in &req.ids {
       operation_logs::create(
           "batch_delete",
           "inbound",
           *id,
           user_id,
       ).await?;
   }
   ```

3. **二次确认**
   - 批量删除需要二次确认
   - 显示影响范围

### 统计数据安全

1. **数据脱敏**
   - 不显示敏感信息
   - 聚合数据展示

2. **访问控制**
   - 用户只能查看自己的统计
   - 管理员可查看全局统计
