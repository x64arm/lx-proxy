## 🔧 技术问题记录

### 问题 1：Xray-core 集成方式

**决策：** 使用 Xray-core 的 JSON 配置方式，通过 Rust 生成配置文件并重启 Xray 服务。

**原因：**
- Xray 官方 Go API 不稳定
- JSON 配置方式成熟可靠
- 便于备份和恢复

### 问题 2：密码哈希算法

**决策：** 使用 Argon2 算法

**原因：**
- 2015 年密码哈希竞赛获胜者
- 抗 GPU/ASIC 攻击
- Rust 生态有成熟实现（argon2 crate）

### 问题 3：Axum 0.8 中间件 API 变更

**问题：** Axum 0.8 中间件签名有变化，`from_fn` 不再接受泛型参数。

**解决：** 
- 将公开路由和受保护路由分开
- 使用 `from_fn_with_state` 为受保护路由添加认证中间件
- 中间件函数签名改为 `Request<Body>` 而非泛型
