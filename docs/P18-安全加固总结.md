## 🔒 安全特性详解

### 1. SQL 注入防护 ✅

**实现方式：**
- 使用 SQLx 参数化查询
- 所有用户输入通过 `.bind()` 传递
- 无字符串拼接 SQL

**代码示例：**
```rust
// ✅ 安全 - 参数化查询
let user = sqlx::query_as::<_, User>(
    r#"SELECT * FROM users WHERE username = $1 LIMIT 1"#
)
.bind(&req.username)
.fetch_optional(&pool)
.await?;

// ❌ 危险 - 字符串拼接（已避免）
// let query = format!("SELECT * FROM users WHERE username = '{}'", username);
```

**验证结果：**
```bash
$ grep -rn "format!(" src/ | grep -E "(SELECT|INSERT|UPDATE|DELETE)"
# (无结果) ✅
```

### 2. XSS 防护 ✅

**实现方式：**
- Vue 3 自动转义所有输出
- 禁用 `v-html`（项目中未使用）
- CSP 响应头限制资源加载

**安全响应头：**
```nginx
Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'; frame-ancestors 'none'
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
```

**验证结果：**
```bash
$ grep -rn "v-html" frontend/src/
# (无结果) ✅
```

---

### 3. CSRF 防护 ✅

**实现方式：**
- JWT Token 存储在内存/LocalStorage
- 每次请求通过 `Authorization: Bearer <token>` 发送
- 浏览器不会自动发送 JWT，天然防 CSRF
- 关键操作二次验证（密码 + TOTP）

**关键操作二次验证：**
- 删除用户
- 删除入站配置
- 更新系统配置
- 批量操作
- 导出数据

**代码示例：**
```rust
// middleware/csrf_protection.rs
pub async fn request_verification(
    State(state): State<AppState>,
    claims: Claims,
    Json(req): Json<VerificationRequest>,
) -> Result<Json<VerificationResponse>, StatusCode> {
    // 1. 验证用户密码
    // 2. 验证 TOTP 码（如果启用）
    // 3. 生成一次性验证令牌（5 分钟有效期）
}
```

### 4. 速率限制 ✅

**实现方式：**
- 基于 IP 地址限流
- 登录接口：5 次/分钟，超限封禁 15 分钟
- API 接口：100 次/分钟，超限封禁 5 分钟
- 内存存储限流状态（生产环境使用 Redis）

**配置：**
```rust
// middleware/rate_limiter.rs
RateLimiterConfig {
    window: Duration::from_secs(60),      // 1 分钟
    max_requests: 100,                    // 100 次请求
    ban_duration: Duration::from_secs(300), // 封禁 5 分钟
}

// 登录更严格
login_config = RateLimiterConfig {
    max_requests: 5,                      // 5 次登录尝试
    ban_duration: Duration::from_secs(900), // 封禁 15 分钟
}
```

**测试：**
```bash
# 快速发送 6 次登录请求
for i in {1..6}; do
  curl -X POST http://localhost/api/auth/login \
    -d '{"username":"admin","password":"wrong"}'
done
# 第 6 次返回 429 Too Many Requests ✅
```

### 5. 敏感数据加密 ✅

**实现方式：**
- AES-256-GCM 加密算法
- 随机 Nonce 保证每次加密结果不同
- 密钥通过环境变量管理

**加密对象：**
- SMTP 密码
- 节点 API 密钥
- 插件敏感配置
- 数据库连接密码（可选）

**代码示例：**
```rust
// crypto/encryptor.rs
let encryptor = DataEncryptor::from_env()?;

// 加密
let encrypted = encryptor.encrypt_string("sensitive-data")?;

// 解密
let decrypted = encryptor.decrypt_string(&encrypted)?;
```

**密钥生成：**
```bash
./scripts/security-setup.sh generate-key
# 输出：ENCRYPTION_KEY=<64-char-hex>
```

### 6. 安全响应头 ✅

**实现的响应头：**

| 响应头 | 值 | 作用 |
|--------|-----|------|
| Content-Security-Policy | default-src 'self' | 防止 XSS 和数据注入 |
| X-Content-Type-Options | nosniff | 防止 MIME 嗅探 |
| X-Frame-Options | DENY | 防止点击劫持 |
| X-XSS-Protection | 1; mode=block | 启用浏览器 XSS 过滤 |
| Referrer-Policy | strict-origin-when-cross-origin | 控制 Referrer 泄露 |
| Permissions-Policy | geolocation=(), microphone=() | 限制浏览器功能 |
| Strict-Transport-Security | max-age=31536000 | 强制 HTTPS |
| Cache-Control | no-store | 防止敏感数据缓存 |

**实现文件：**
```
backend/src/middleware/security_headers.rs
```

### 7. HTTPS 强制启用 ✅

**实现方式：**
- Nginx 反向代理强制 HTTPS
- Let's Encrypt 免费 SSL 证书
- HTTP 自动跳转 HTTPS（301）
- HSTS 启用（1 年有效期）

**部署文档：**
```
docs/HTTPS 部署配置.md
```

**快速部署：**
```bash
# 1. 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 2. 获取证书
sudo certbot --nginx -d your-domain.com

# 3. 自动续期
sudo certbot renew --dry-run
```
