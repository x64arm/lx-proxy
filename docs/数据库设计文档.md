## 📋 表详细设计

### 1. users - 用户账户表

**用途：** 存储系统用户账户信息

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(255),
    role VARCHAR(20) NOT NULL DEFAULT 'user',
    enabled BOOLEAN NOT NULL DEFAULT true,
    last_login_at TIMESTAMPTZ,
    last_login_ip INET,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**字段说明：**
- `id`: UUID 主键
- `username`: 用户名，唯一，3-50 字符
- `password_hash`: Argon2id 密码哈希
- `email`: 邮箱地址，用于通知
- `role`: 角色（admin/user）
- `enabled`: 账户是否启用
- `last_login_at`: 最后登录时间
- `last_login_ip`: 最后登录 IP
- `created_at`: 创建时间
- `updated_at`: 更新时间

**索引：**
```sql
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email) WHERE email IS NOT NULL;
CREATE INDEX idx_users_role ON users(role);
CREATE INDEX idx_users_enabled ON users(enabled);
CREATE INDEX idx_users_created_at ON users(created_at DESC);
```

### 2. inbound_configs - 入站配置表

**用途：** 存储 Xray 入站代理配置

```sql
CREATE TABLE inbound_configs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE SET NULL,
    tag VARCHAR(100) UNIQUE NOT NULL,
    protocol VARCHAR(50) NOT NULL,
    port INTEGER NOT NULL,
    settings JSONB NOT NULL DEFAULT '{}',
    stream_settings JSONB DEFAULT '{}',
    sniffing JSONB DEFAULT '{}',
    enable BOOLEAN NOT NULL DEFAULT true,
    traffic_used BIGINT NOT NULL DEFAULT 0,
    traffic_limit BIGINT,
    expire_at TIMESTAMPTZ,
    ip_limit INTEGER,
    total_upload BIGINT NOT NULL DEFAULT 0,
    total_download BIGINT NOT NULL DEFAULT 0,
    reset_day INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**字段说明：**
- `id`: UUID 主键
- `user_id`: 关联用户（可空）
- `tag`: 配置标签，唯一标识
- `protocol`: 协议类型（vmess/vless/trojan/shadowsocks）
- `port`: 监听端口
- `settings`: 协议设置（JSON）
- `stream_settings`: 流控设置（JSON）
- `sniffing`: 嗅探设置（JSON）
- `enable`: 是否启用
- `traffic_used`: 已用流量（字节）
- `traffic_limit`: 流量限制（字节，NULL 为不限）
- `expire_at`: 过期时间
- `ip_limit`: IP 连接数限制
- `total_upload`: 累计上传
- `total_download`: 累计下载
- `reset_day`: 流量重置日（0-28）

**索引：**
```sql
CREATE INDEX idx_inbounds_user_id ON inbound_configs(user_id);
CREATE INDEX idx_inbounds_protocol ON inbound_configs(protocol);
CREATE INDEX idx_inbounds_enable ON inbound_configs(enable);
CREATE INDEX idx_inbounds_expire ON inbound_configs(expire_at) WHERE enable = true;
CREATE INDEX idx_inbounds_traffic ON inbound_configs(traffic_used, traffic_limit) WHERE traffic_limit IS NOT NULL;
CREATE INDEX idx_inbounds_created_at ON inbound_configs(created_at DESC);
```

### 3. traffic_logs - 流量日志表

**用途：** 记录每小时流量使用情况

```sql
CREATE TABLE traffic_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    inbound_id UUID NOT NULL REFERENCES inbound_configs(id) ON DELETE CASCADE,
    upload BIGINT NOT NULL DEFAULT 0,
    download BIGINT NOT NULL DEFAULT 0,
    recorded_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**分区表设计（可选，大数据量时）：**
```sql
-- 按月分区
CREATE TABLE traffic_logs_2026_03 PARTITION OF traffic_logs
    FOR VALUES FROM ('2026-03-01') TO ('2026-04-01');
```

**索引：**
```sql
CREATE INDEX idx_traffic_inbound_id ON traffic_logs(inbound_id);
CREATE INDEX idx_traffic_recorded_at ON traffic_logs(recorded_at DESC);
CREATE INDEX idx_traffic_inbound_date ON traffic_logs(inbound_id, recorded_at DESC);
```

**维护：**
```sql
-- 清理 90 天前的日志
DELETE FROM traffic_logs WHERE recorded_at < NOW() - INTERVAL '90 days';
```

## 📈 性能优化

### 1. 分区表（traffic_logs）

```sql
-- 创建分区表
CREATE TABLE traffic_logs (
    id UUID,
    inbound_id UUID NOT NULL,
    upload BIGINT NOT NULL DEFAULT 0,
    download BIGINT NOT NULL DEFAULT 0,
    recorded_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
) PARTITION BY RANGE (recorded_at);

-- 创建月分区
CREATE TABLE traffic_logs_2026_03 PARTITION OF traffic_logs
    FOR VALUES FROM ('2026-03-01') TO ('2026-04-01');

CREATE TABLE traffic_logs_2026_04 PARTITION OF traffic_logs
    FOR VALUES FROM ('2026-04-01') TO ('2026-05-01');
```

### 2. 物化视图（流量统计）

```sql
CREATE MATERIALIZED VIEW mv_daily_traffic AS
SELECT 
    DATE(recorded_at) as date,
    inbound_id,
    SUM(upload) as total_upload,
    SUM(download) as total_download
FROM traffic_logs
GROUP BY DATE(recorded_at), inbound_id;

-- 刷新视图
REFRESH MATERIALIZED VIEW mv_daily_traffic;
```

### 3. 定期维护

```sql
-- 清理过期日志
DELETE FROM traffic_logs WHERE recorded_at < NOW() - INTERVAL '90 days';
DELETE FROM operation_logs WHERE created_at < NOW() - INTERVAL '180 days';
DELETE FROM email_logs WHERE created_at < NOW() - INTERVAL '90 days';

-- 清理过期会话
DELETE FROM login_sessions WHERE expires_at < NOW();

-- 分析表
ANALYZE users;
ANALYZE inbound_configs;
ANALYZE traffic_logs;
```

## 🔐 安全考虑

### 1. 敏感数据加密

```sql
-- 使用 pgcrypto 扩展
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- 加密存储
UPDATE users SET password_hash = crypt('password', gen_salt('argon2'));
```

### 2. 行级安全（RLS）

```sql
-- 启用 RLS
ALTER TABLE inbound_configs ENABLE ROW LEVEL SECURITY;

-- 创建策略：用户只能查看自己的配置
CREATE POLICY user_inbound_policy ON inbound_configs
    FOR SELECT
    USING (user_id = current_setting('app.current_user_id')::uuid);
```

### 3. 审计日志

所有敏感操作都应记录到 `operation_logs` 表。
