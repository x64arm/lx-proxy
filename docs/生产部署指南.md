## 📋 部署前检查

### 系统要求

**最低配置：**
- CPU: 1 核
- 内存：512MB
- 磁盘：10GB

**推荐配置：**
- CPU: 2 核+
- 内存：2GB+
- 磁盘：50GB+ SSD
- 系统：Ubuntu 22.04 LTS

### 端口要求

| 端口 | 用途 | 协议 |
|------|------|------|
| 80 | HTTP (重定向) | TCP |
| 443 | HTTPS | TCP |
| 8080 | 管理面板 | TCP |
| 5432 | PostgreSQL | TCP |
| 62780 | Xray API | TCP |
| 10000+ | 代理端口 | TCP/UDP |

## 🔒 HTTPS 配置

### 使用 Let's Encrypt 免费证书

**步骤：**

1. **安装 Certbot**
   ```bash
   apt-get update
   apt-get install certbot python3-certbot-nginx
   ```

2. **获取证书**
   ```bash
   certbot --nginx -d your-domain.com
   ```

3. **配置 Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       return 301 https://$server_name$request_uri;
   }

   server {
       listen 443 ssl http2;
       server_name your-domain.com;

       ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
       ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

       ssl_protocols TLSv1.2 TLSv1.3;
       ssl_ciphers HIGH:!aNULL:!MD5;
       ssl_prefer_server_ciphers on;

       location / {
           proxy_pass http://127.0.0.1:8080;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

4. **自动续期**
   ```bash
   # 添加定时任务
   crontab -e
   
   # 每天凌晨 2 点检查续期
   0 2 * * * certbot renew --quiet
   ```

## 🛡️ 安全加固

### 1. 防火墙配置

**UFW（Ubuntu）：**
```bash
# 安装 UFW
apt-get install ufw

# 允许 SSH
ufw allow 22/tcp

# 允许 HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# 允许管理面板（限制 IP）
ufw allow from 192.168.1.0/24 to any port 8080

# 启用防火墙
ufw enable

# 查看状态
ufw status
```

**Firewalld（CentOS）：**
```bash
# 启动服务
systemctl start firewalld
systemctl enable firewalld

# 开放端口
firewall-cmd --permanent --add-service=ssh
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --permanent --add-port=8080/tcp

# 重载配置
firewall-cmd --reload
```

### 2. 修改默认端口

**SSH 端口：**
```bash
# 编辑 SSH 配置
vim /etc/ssh/sshd_config

# 修改端口
Port 2222

# 重启 SSH
systemctl restart sshd
```

**面板端口：**
```bash
# 编辑 .env
PORT=8443

# 重启服务
docker-compose restart
```

### 3. Fail2Ban 防护

**安装配置：**
```bash
# 安装
apt-get install fail2ban

# 创建配置
cat > /etc/fail2ban/jail.local << EOF
[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 3600

[lx-proxy]
enabled = true
port = http,https
filter = lx-proxy
logpath = /var/log/lx-proxy/*.log
maxretry = 5
bantime = 7200
EOF

# 重启服务
systemctl restart fail2ban
```

### 4. 数据库安全

**PostgreSQL 加固：**
```sql
-- 修改默认密码
ALTER USER postgres WITH PASSWORD 'strong-password';

-- 限制远程访问
ALTER SYSTEM SET listen_addresses = 'localhost';

-- 创建专用用户
CREATE USER lx_proxy WITH PASSWORD 'strong-password';
GRANT ALL PRIVILEGES ON DATABASE lx_proxy TO lx_proxy;
```

## 🔄 备份策略

### 数据库备份

**自动备份脚本：**
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/backups/lx-proxy"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="lx_proxy"
DB_USER="postgres"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -U $DB_USER $DB_NAME | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 保留最近 7 天备份
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +7 -delete

# 上传到云存储（可选）
# aws s3 cp $BACKUP_DIR/db_$DATE.sql.gz s3://your-bucket/backups/
```

**定时任务：**
```bash
# 每天凌晨 3 点备份
crontab -e
0 3 * * * /path/to/backup.sh
```

### 配置文件备份

```bash
# 备份重要配置
tar -czf config_backup_$(date +%Y%m%d).tar.gz \
    .env \
    backend/.env \
    docker-compose.yml \
    nginx.conf
```

## 🔧 故障排查

### 常见问题

**1. 服务无法启动**
```bash
# 查看日志
docker-compose logs backend

# 检查端口占用
netstat -tlnp | grep 8080

# 检查环境变量
docker-compose config
```

**2. 数据库连接失败**
```bash
# 检查数据库状态
docker-compose ps db

# 测试连接
docker-compose exec backend \
    psql postgresql://user:pass@db:5432/lx_proxy
```

**3. Xray 无法启动**
```bash
# 查看 Xray 日志
docker-compose logs xray

# 检查配置文件
docker-compose exec backend cat /app/xray_config.json

# 测试配置
docker-compose exec xray xray test -config /etc/xray/config.json
```
