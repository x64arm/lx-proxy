## 🚀 快速入门

### 1. 登录系统

1. 打开浏览器访问：`http://your-server-ip`
2. 输入默认凭据：
   - 用户名：`admin`
   - 密码：`admin123`
3. **首次登录后请立即修改密码！**

### 2. 创建第一个用户

1. 进入「用户管理」页面
2. 点击「创建用户」
3. 填写用户名和密码
4. 选择角色（普通用户/管理员）
5. 点击「保存」

### 3. 创建入站配置

1. 进入「入站配置」页面
2. 点击「创建入站」
3. 选择协议（推荐 Vmess 或 Vless）
4. 配置端口和参数
5. 设置流量限制和过期时间
6. 点击「保存」

### 4. 生成订阅链接

1. 进入「订阅链接」页面
2. 选择要分享的入站配置
3. 复制订阅链接或扫描二维码
4. 在客户端中导入订阅

## 👥 用户管理

### 创建用户

**步骤：**
1. 导航到「用户管理」
2. 点击「创建用户」按钮
3. 填写信息：
   - **用户名**：3-20 个字符
   - **密码**：至少 6 个字符
   - **角色**：admin（管理员）或 user（普通用户）
   - **邮箱**：用于接收通知（可选）

**权限说明：**
- **管理员**：可以管理所有用户和配置
- **普通用户**：只能查看和管理自己的配置

### 编辑用户

1. 在用户列表中找到目标用户
2. 点击「编辑」按钮
3. 修改信息
4. 点击「保存」

### 删除用户

⚠️ **警告：** 删除用户会同时删除其所有入站配置！

1. 点击用户行的「删除」按钮
2. 确认删除操作

## 📡 入站配置

### 支持的协议

| 协议 | 特点 | 推荐场景 |
|------|------|----------|
| **Vmess** | 加密传输，抗封锁 | 日常使用 |
| **Vless** | 轻量，高性能 | 高性能需求 |
| **Trojan** | 伪装 HTTPS，隐蔽性好 | 严格网络环境 |
| **Shadowsocks** | 简单快速 | 基础代理需求 |

### 创建入站配置

**基本配置：**
1. **标签**：配置的唯一标识（如：vmess-001）
2. **协议**：选择代理协议
3. **端口**：监听端口（建议 1024-65535）
4. **关联用户**：选择配置所属用户

**高级配置：**
1. **流量限制**：设置每月流量上限（字节）
2. **过期时间**：配置自动失效时间
3. **IP 限制**：限制同时连接的 IP 数

**传输配置：**
- **TCP**：基础传输，兼容性好
- **WS**：WebSocket，可伪装成网站
- **mKCP**：KCP 协议，适合高延迟网络
- **QUIC**：基于 UDP，低延迟

### 编辑入站

1. 点击配置的「编辑」按钮
2. 修改参数
3. 点击「保存」
4. 系统会自动重启 Xray 应用配置

### 重置流量

当用户流量用尽时：
1. 找到对应配置
2. 点击「重置流量」按钮
3. 流量计数归零，服务恢复

### 启用/禁用配置

- 点击开关按钮可快速启用/禁用配置
- 禁用的配置不会出现在订阅链接中

## 📊 流量统计

### 查看流量使用

**仪表盘：**
- 总流量使用
- 活跃用户数
- 系统资源使用率

**流量统计页面：**
1. 选择时间范围（最近 7 天/30 天/自定义）
2. 查看上传/下载流量图表
3. 按入站配置筛选

### 流量告警

系统会在以下情况发送邮件告警：
- 流量使用达到 70%（黄色告警）
- 流量使用达到 90%（红色告警）
- 流量用尽（服务自动禁用）

### 流量重置

**自动重置：**
- 可在系统设置中配置每月自动重置日期
- 默认每月 1 日凌晨 2 点执行

**手动重置：**
- 在入站配置页面点击「重置流量」

## ⚙️ 系统设置

### 基本设置

- **网站标题**：面板显示的名称
- **语言**：界面语言（中文/English）
- **时区**：服务器时区设置

### Xray 设置

- **Xray 路径**：Xray 可执行文件位置
- **配置文件路径**：Xray 配置文件位置
- **日志级别**：debug/info/warning/error

### 流量设置

- **每月流量重置日**：自动重置日期
- **统计保留天数**：流量记录保存时长
- **告警阈值**：触发告警的百分比

### 安全设置

- **会话超时**：登录有效期（小时）
- **最大登录尝试**：密码错误锁定阈值
- **IP 封禁时长**：锁定持续时间（分钟）
- **双因素认证**：启用/禁用 TOTP

### 通知设置

**SMTP 配置：**
1. SMTP 服务器：如 smtp.gmail.com
2. SMTP 端口：587（TLS）或 465（SSL）
3. 用户名：邮箱地址
4. 密码：应用专用密码
5. 发件人：显示名称

**测试邮件：**
- 输入测试邮箱
- 点击「发送测试邮件」
- 检查是否收到

### 系统信息

实时显示：
- CPU 使用率
- 内存使用
- 磁盘空间
- Xray 运行状态
- 数据库连接数

## 🔐 双因素认证

### 启用 TOTP

**步骤：**
1. 进入「安全中心」→「双因素认证」
2. 点击「生成 TOTP 密钥」
3. 使用 Authenticator 应用扫描二维码：
   - Google Authenticator
   - Microsoft Authenticator
   - Authy
4. 输入 6 位验证码
5. 点击「验证并启用」

### 备用代码

⚠️ **重要：** 请妥善保存备用代码！

- 系统生成 10 个备用代码
- 每个代码只能使用一次
- 用于手机丢失时登录
- 建议打印或保存到安全位置

### 禁用 TOTP

1. 进入双因素认证页面
2. 点击「禁用双因素认证」
3. 输入密码确认
4. TOTP 被禁用

## 📧 邮件通知

### 配置邮件

1. 进入「通知设置」→「邮件通知」
2. 填写 SMTP 配置
3. 点击「发送测试邮件」验证
4. 点击「保存配置」

### 通知类型

系统会自动发送以下邮件：

**流量告警：**
- 使用量达 70%：提醒注意
- 使用量达 90%：紧急告警

**到期提醒：**
- 到期前 7 天：重要提醒
- 到期前 3 天：紧急提醒
- 到期当天：最后通知

**禁用通知：**
- 流量用尽自动禁用
- 配置过期自动禁用

## ❓ 常见问题

### Q1: 忘记密码怎么办？

**解决方法：**
1. 如果有备用邮箱，使用邮件重置
2. 联系管理员重置密码
3. 服务器命令行执行：
   ```bash
   make reset-admin
   ```

### Q2: 如何修改默认密码？

**步骤：**
1. 登录后进入「系统设置」
2. 找到「安全设置」
3. 修改密码并保存

### Q3: 流量统计不准确？

**可能原因：**
- Xray API 未正确配置
- 定时任务未运行
- 数据库连接问题

**解决方法：**
1. 检查 Xray API 配置
2. 查看后端日志
3. 重启 LX-Proxy 服务

### Q4: 如何启用 HTTPS？

**步骤：**
1. 获取 SSL 证书（Let's Encrypt 或购买）
2. 配置 Nginx 反向代理
3. 参考「部署指南」中的 HTTPS 配置

### Q5: 订阅链接无法使用？

**检查项目：**
- 服务器防火墙是否开放端口
- 入站配置是否启用
- 订阅链接是否正确复制
- 客户端配置是否正确

### Q6: 如何备份数据？

**数据库备份：**
```bash
# 手动备份
make backup

# 自动备份（启用 backup profile）
docker-compose --profile backup up -d
```

备份文件位置：`./data/backups/`

### Q7: 如何更新 LX-Proxy？

**自动更新：**
```bash
make update
```

**手动更新：**
```bash
git pull
docker-compose down
docker-compose up -d
```
