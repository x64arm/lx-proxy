### 1️⃣ 双因素认证（TOTP）

**后端文件：**
- `backend/src/totp/mod.rs` - TOTP 核心模块
- `backend/src/handlers/totp.rs` - TOTP API 处理器
- `backend/migrations/003_add_totp_support.sql` - 数据库迁移

**前端文件：**
- `frontend/src/views/TotpSettingsView.vue` - TOTP 设置页面

**功能特性：**
- ✅ 生成 TOTP 密钥（Base32 编码）
- ✅ 生成 QR Code 二维码 URL
- ✅ 支持 Google/Microsoft Authenticator
- ✅ 6 位验证码验证
- ✅ 10 个备用代码生成
- ✅ 启用/禁用 TOTP
- ✅ 备用代码登录支持

**API 端点：**
```bash
# 初始化 TOTP 设置
POST /api/totp/setup
{
  "user_id": "uuid"
}

# 验证并启用
POST /api/totp/{user_id}/verify
{
  "code": "123456"
}

# 禁用 TOTP
POST /api/totp/{user_id}/disable
{
  "password": "user-password"
}

# 获取状态
GET /api/totp/{user_id}/status

# 备用代码登录
POST /api/totp/backup-login
{
  "user_id": "uuid",
  "backup_code": "12345678"
}
```

**数据库表：**
```sql
user_totp_configs
- id: UUID
- user_id: UUID (unique)
- secret: VARCHAR(255)
- enabled: BOOLEAN
- verified: BOOLEAN
- backup_codes: JSONB
- created_at: TIMESTAMPTZ
- updated_at: TIMESTAMPTZ
```

## 🎯 使用流程

### TOTP 设置流程
1. 用户进入 TOTP 设置页面
2. 点击"生成 TOTP 密钥"
3. 使用 Authenticator 扫描二维码或手动输入密钥
4. 输入 6 位验证码验证
5. 保存备用代码到安全位置
6. TOTP 启用成功

### 邮件配置流程
1. 进入邮件设置页面
2. 填写 SMTP 服务器信息
3. 输入测试邮箱地址
4. 发送测试邮件验证配置
5. 保存配置

## 📝 注意事项

### TOTP 相关
1. **时间同步** - 确保服务器时间准确（使用 NTP）
2. **备用代码** - 提醒用户妥善保存，一旦丢失无法恢复
3. **宽限期** - 可考虑实现±1 个周期的宽限期

### 邮件相关
1. **应用专用密码** - Gmail/Outlook 需使用应用专用密码
2. **防火墙** - 确保 SMTP 端口（587/465）未被阻止
3. **频率限制** - 建议实现邮件发送频率限制
