## 🔒 实现的安全特性

### 1. 安全响应头（7 种）

```rust
// middleware/security_headers.rs
- Content-Security-Policy: 防止 XSS 和数据注入
- X-Content-Type-Options: nosniff
- X-Frame-Options: DENY（防止点击劫持）
- X-XSS-Protection: 1; mode=block
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy: 限制浏览器功能
- Cache-Control: no-store（API 响应）
- Strict-Transport-Security: max-age=31536000（HTTPS 强制）
```

### 2. CSRF 防护与二次验证

```rust
// middleware/csrf_protection.rs
敏感操作列表：
- DELETE_USER（删除用户）
- DELETE_INBOUND（删除入站）
- UPDATE_CONFIG（更新配置）
- UPDATE_XRAY_CONFIG（更新 Xray 配置）
- DISABLE_TOTP（禁用双因素）
- BATCH_DELETE（批量删除）
- BATCH_RESET（批量重置）
- EXPORT_DATA（导出数据）

验证流程：
1. 用户输入密码
2. 验证 TOTP 码（如果启用）
3. 生成 5 分钟有效期的验证令牌
4. 记录安全审计日志
```

### 3. 速率限制（防暴力破解）

```rust
// middleware/rate_limiter.rs
API 接口：100 次/分钟 → 超限封禁 5 分钟
登录接口：5 次/分钟 → 超限封禁 15 分钟
基于 IP 地址限流
```

### 4. 数据加密（AES-256-GCM）

```rust
// crypto/encryptor.rs
加密算法：AES-256-GCM
密钥长度：256 位（32 字节）
Nonce：随机生成（12 字节）
加密对象：SMTP 密码、API 密钥、敏感配置
```

### 5. 安全审计日志

```rust
// audit/service.rs
log_security_event() 函数：
- 记录用户 ID
- 事件类型（登录失败、验证失败等）
- 事件描述
- IP 地址
- 时间戳
```

## 📝 部署指南

### 1. 生成密钥

```bash
cd /root/.openclaw/workspace/lx-proxy

# 生成加密密钥
./scripts/security-setup.sh generate-key

# 生成 JWT 密钥
./scripts/security-setup.sh generate-jwt

# 将密钥添加到 .env 文件
echo "ENCRYPTION_KEY=<生成的密钥>" >> .env
echo "JWT_SECRET=<生成的密钥>" >> .env
```

### 2. 配置 HTTPS

参考 `docs/HTTPS 部署配置.md`：

```bash
# 方法 1：Certbot（推荐）
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com

# 方法 2：Nginx Proxy Manager（Docker）
# 通过 Web 界面自动申请证书
```

### 3. 安全检查

```bash
# 运行完整安全检查
./scripts/security-setup.sh check-security

# 检查依赖漏洞
./scripts/security-setup.sh check-deps
```

### 4. 重启服务

```bash
# Docker 部署
docker-compose restart backend

# 或手动部署
make restart
```
